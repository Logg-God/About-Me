Technical Architecture

This project uses a service-based architecture where each system owns one responsibility. This supports scaling and collaboration.

Services

  • CombatService: validates hits and applies damage (server-authoritative)
  • EnemyService: spawning, AI state, health, death
  • DungeonService: floor progression, doors, boss triggers
  • LootService: drop tables, rarity rolls, rewards
  • DataService: save/load coins and unlocks

Client vs Server

  • Client: input, animations, UI, camera feedback
  • Server: validation, damage, enemy health, rewards, progression

Source control + collaboration

  • GitHub repo with branches and pull-request style reviews
  • Freelancers work on feature branches (UI/VFX) without touching core systems
