Project Overview + MVP

What I’m building, why it’s fun, and the exact MVP scope.

Project Overview

Elevator pitch: Dungeon Fighter is a floor-based PvE dungeon game where players fight through rooms, defeat a boss, and progress to the next level.

Core gameplay loop

  • Spawn → Explore → Fight enemies → Earn loot/coins → Boss fight → Next floor

MVP (Minimum Viable Product)

The MVP is a vertical slice: the smallest complete version that proves the core loop is fun. It’s designed to be shippable quickly and expandable without rewriting core systems.

MVP goals

  • One complete floor experience: start → combat → boss → reward
  • Server-authoritative damage (anti-exploit)
  • Modular service structure for scaling and collaboration

MVP feature list

  • 1 floor (single run loop)
  • 1 weapon: sword
  • 2 enemies: melee chaser + tank bruiser
  • 1 boss: 2 attacks + simple phase change
  • UI: health bar + objective (“Defeat the boss”)
  • Rewards: coins or a basic loot drop
  • Fail state: death → restart at floor start

After MVP ships, I’ll expand into a full 3-floor dungeon (Release 1) and then replayability systems (Release 2). Details are on the Roadmap page.
