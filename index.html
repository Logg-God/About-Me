Project Overview

Elevator pitch: Enter a dungeon floor, defeat enemies, find upgrades, and beat a boss to progress.

Core gameplay loop

  • Spawn → Explore rooms → Fight enemies → Earn loot/coins → Boss fight → Next floor

Current status: Planning (MVP build in progress). This page will update daily with devlog entries and media.

MVP (Minimum Viable Product)

The MVP is a vertical slice: the smallest complete version that proves the core loop is fun.

MVP goals

  • One complete floor experience: start → combat → boss → reward
  • Clean, modular code structure (services) to support later releases
  • Server-authoritative damage to reduce exploit risk

MVP feature list

  • 1 floor (single run loop)
  • 1 weapon: sword
  • 2 enemies: melee chaser + tank bruiser
  • 1 boss: 2 attacks + simple phase change
  • UI: health bar + objective (“Defeat the boss”)
  • Rewards: coins or a basic loot drop
  • Fail state: death → restart at floor start

Release Roadmap

After MVP, I’ll build two releases: content expansion, then replayability systems.

Release 1 — “3-floor dungeon”

  • 3 floors fully playable
  • 2–3 weapons (sword + heavy + ranged)
  • 1 new enemy type (ranged)
  • Shop or rest area between floors
  • Basic data persistence (coins)

Release 2 — “Replayability”

  • Room variation (template-based)
  • Run modifiers (risk/reward)
  • Improved loot rarity + perks
  • Polish pass: VFX/SFX + better UI

Technical Architecture

The project uses a modular service-based architecture where each gameplay system has a single responsibility. This makes the codebase easier to scale, test, and collaborate on.

High-level services

  • CombatService: validates hits and applies damage (server-authoritative)
  • EnemyService: spawns enemies, AI state, health, death
  • DungeonService: run flow, floor progression, boss triggers
  • LootService: drop tables, rarity rolls, rewards
  • DataService: save/load coins and unlocks

Client vs Server responsibilities

  • Client: input, animations, UI, camera feedback
  • Server: damage validation, enemy health, rewards, progression

Source control: GitHub + branches + PR reviews for collaboration.

Generative AI Workflow

I use AI as a productivity tool for scaffolding modules, generating test cases, summarizing bugs, and refactoring for readability. I validate everything through in-game testing, logging, and code review.

How AI fits into my process

  • Generate edge cases and testing checklists
  • Refactor scripts into cleaner functions
  • Translate design goals into implementable tasks
