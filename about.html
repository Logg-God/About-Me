About Me

Who I am, what I build, and why I’m building Dungeon Fighter.

Background

I’m Logan Hampel, a 17-year-old (11th grade) developer. I’ve played Roblox since age 8 and have been scripting in Luau since age 12. I focus on engineering: clean architecture, iteration, and shipping working systems.

Skills

  • Luau (Roblox), client/server architecture, RemoteEvents/Functions
  • Gameplay systems: combat, enemy AI, progression, UI
  • Source control: GitHub commits, branches, PR-style reviews

Why this project

Dungeon Fighter is designed to show practical CS skills in a real system: decomposition, modular design, validation and safety (server authority), and iteration from MVP to releases.

What I’m optimizing for

  • Fun first: tight combat feel + readable enemy patterns
  • Server authority: validate hits/damage on the server to reduce exploits
  • Scalability: modular services so multiple people can contribute safely

Collaboration

  • Code review and feedback from mentors (including my dad)
  • Outsource time-sinks (UI polish, VFX/SFX) while keeping core systems in-house
