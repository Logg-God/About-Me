Roadmap

Release Roadmap

MVP → Release 1 (content) → Release 2 (replayability), plus a 7-day sprint plan.

Release 1 — “3-floor dungeon”

After MVP, expand the content while keeping architecture stable.

  • 3 floors fully playable
  • 2–3 weapons (sword + heavy + ranged)
  • 1 new enemy type (ranged)
  • Shop/rest area between floors
  • Basic data persistence (coins)

Release 2 — “Replayability”

Add systems that make the game worth repeating.

  • Room variation (template-based)
  • Run modifiers (risk/reward)
  • Improved loot rarity + perks
  • Polish pass: VFX/SFX + better UI

7-Day Build Plan (Sprint)

  • Day 1: Website + repo setup + project scaffold
  • Day 2: Combat baseline (server-authoritative) + first enemy prototype
  • Day 3: Enemy AI loop + player health + basic UI
  • Day 4: Boss fight + arena + reward drop
  • Day 5: Dungeon flow polish (doors/objectives/checkpoints)
  • Day 6: Website updates: architecture write-up + AI workflow + screenshots
  • Day 7: Bug bash + balancing + record demo clip

Outsourcing plan (if needed)

With a $500 budget, I’ll outsource time-sinks (UI polish, VFX/SFX, environment art) while keeping core systems (combat, AI, dungeon flow, data) in-house to preserve ownership and learning.
